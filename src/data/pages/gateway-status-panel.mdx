# 服务状态与连接排查

当客户端连接失败时，不一定是 API Key 错误。可以按“本地配置 → 网络连接 → 模型服务 → 账户额度”的顺序排查。

## 先确认四件事

1. Base URL 是否为 `https://www.aigcwuji.com`。
2. API Key 是否完整，没有多复制空格或换行。
3. 模型名是否来自账户页面展示的 model id。
4. 修改配置后，是否已经完全关闭客户端和终端再重新打开。

## 常见情况

| 现象 | 优先检查 |
|---|---|
| 401 / Unauthorized | API Key、字段名、配置文件路径 |
| timeout / connection error | 网络、代理、Base URL、DNS |
| model not found | 模型名、provider、账户权限 |
| 429 / rate limit | 并发、余额、频率限制、重试策略 |
| 修改后没变化 | 旧进程、旧终端、改错配置文件 |

## 使用建议

如果多个客户端同时异常，先暂停频繁重试，避免消耗额度或触发限流。可以等待几分钟后，用最短测试消息复测。
