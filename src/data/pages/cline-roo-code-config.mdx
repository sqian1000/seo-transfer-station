# Cline / Roo Code 配置入口

Cline、Roo Code 这类 VS Code / Cursor 扩展通常不读取 Claude Code 的 `ANTHROPIC_BASE_URL`，也不读取 Codex 的 `~/.codex/config.toml`。它们更常见的接入方式是：在扩展设置里选择 **OpenAI Compatible** provider，然后填写 Base URL、API Key 和 Model ID。

## 推荐配置

| 字段 | 推荐值 | 说明 |
|---|---|---|
| Provider / API Provider | `OpenAI Compatible` | 不选官方 OpenAI 专用项，除非你明确要直连官方 OpenAI |
| Base URL | `https://www.aigcwuji.com/v1` | OpenAI 兼容接口通常带 `/v1` |
| API Key | 账户页面生成的项目 Key | 不要提交到 Git、截图或公开文档 |
| Model / Model ID | 账户页面展示的 model id | 不猜模型名，不复制其他平台教程里的名称 |

## Cline 配置步骤

1. 打开 VS Code / Cursor 里的 Cline 面板。
2. 点击设置齿轮。
3. API Provider 选择 **OpenAI Compatible**。
4. Base URL 填：

```text
https://www.aigcwuji.com/v1
```

5. API Key 填账户页面生成的 Key。
6. Model / Model ID 填账户页面展示的模型名。
7. 点击 Verify / 测试连接；如果没有按钮，就发送一句短消息：

```text
请回复“配置已生效”。
```

Cline 官方 OpenAI Compatible 文档强调，这个 provider 需要填 Base URL、API Key、Model ID；Base URL 不是 `https://api.openai.com/v1`，而是你所选兼容服务商提供的地址。

## Roo Code 配置步骤

1. 打开 VS Code / Cursor 里的 Roo Code 面板。
2. 进入 Roo Code settings。
3. API Provider 选择 **OpenAI Compatible**。
4. Base URL 填：

```text
https://www.aigcwuji.com/v1
```

5. API Key 填账户页面生成的 Key。
6. Model 选择或输入账户页面展示的模型名。
7. 发送短消息验证；如果出现工具调用错误，先换一个明确支持 tool calling 的模型。

Roo Code 官方文档特别说明：Roo Code 只使用 OpenAI native tool calling，没有 XML fallback。因此模型不仅要能聊天，还要支持 OpenAI-compatible function/tool calling；否则会出现工具调用失败、无法读写文件或任务中断。

## 验证顺序

建议按这个顺序排查，别一上来就换模型：

1. 当前面板确认是 Cline 或 Roo Code，不是 Cursor 原生 AI、Claude Code 或 Codex。
2. Provider 确认选中 **OpenAI Compatible**。
3. Base URL 确认是 `https://www.aigcwuji.com/v1`，没有多余空格。
4. API Key 确认来自账户页面，且没有复制丢字符。
5. Model ID 确认和账户页面完全一致。
6. 完全重启编辑器后再测一次，避免旧进程缓存。
7. Roo Code 额外确认模型支持 OpenAI native tool calling。

## 常见排错

| 现象 | 可能原因 | 处理方式 |
|---|---|---|
| `Invalid API Key` / 401 | Key 错、复制不完整、填到了错误扩展 | 重新生成 Key，并确认填在 Cline/Roo Code 当前 profile |
| `Model Not Found` | 模型名不在账户权限内，或填了官方模型别名 | 回账户页面复制完整 model id |
| 连接失败 | Base URL 缺 `/v1`、网络不可达、服务地址填错 | 改为 `https://www.aigcwuji.com/v1` 后重启扩展 |
| Cline 能聊但效果异常 | 模型参数或上下文窗口设置不匹配 | 先恢复默认高级参数，只保留 provider/base/key/model |
| Roo Code 工具调用失败 | 模型不支持 OpenAI native tool calling，或兼容接口 tool calling 实现不完整 | 切换到支持 tool calling 的模型，再测文件读取/编辑 |
| 改完没反应 | 扩展仍在读旧 profile 或旧进程 | 完全退出 VS Code / Cursor 后重新打开 |

## 和其他插件的区别

- Cursor 原生 AI：看 [Cursor API 配置入口](/cursor-api-config)。
- Claude Code 扩展：看 `~/.claude/settings.json`、Claude Code env 和 IDE Extension settings。
- Codex 扩展：看 `~/.codex/config.toml`、`~/.codex/auth.json`。
- Cline / Roo Code：看扩展内 **OpenAI Compatible** provider 表单。

如果你在同一个 Cursor 里装了多个 AI 扩展，排错时先确认当前报错来自哪个面板。不同扩展的登录态和配置文件不会自动同步。
