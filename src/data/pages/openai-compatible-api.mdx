# OpenAI 兼容 API Base URL 总指南

OpenAI compatible API 的核心，是让客户端把请求发到自定义 `base_url`，并继续使用 Bearer Token、`model` 字段和兼容协议。Codex、Cursor、Cline、Roo Code 和很多 SDK 都属于这一类接入场景。

## 通用字段

| 字段 | 说明 |
|---|---|
| `base_url` / `openai_base_url` | 示例：`https://www.531288.xyz/v1` 或账户页面显示的兼容入口 |
| `OPENAI_API_KEY` | 账户页面生成的 API Key |
| `model` | 以账户页面展示为准，不猜测官方 model id |
| `timeout` | 建议生产环境显式设置超时 |
| `retry` | 对 429、5xx 设置有限次数的退避重试 |

本文统一使用自有中转地址 `https://www.531288.xyz`。OpenAI 兼容 SDK 通常填写 `https://www.531288.xyz/v1`。

## 最小 SDK 示例

```ts
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: "https://www.531288.xyz/v1",
});

const res = await client.chat.completions.create({
  model: "账户页面展示的模型名",
  messages: [{ role: "user", content: "请回复配置已生效" }],
});
```

## Cursor / Cline / Roo Code 通用检查

1. Provider 选择 OpenAI compatible 或自定义兼容接口。
2. Base URL 填`https://www.531288.xyz/v1`。
3. API Key 使用当前项目 Key，不复用高权限主 Key。
4. Model 使用账户页面模型列表展示值。
5. 出现 401、404、429 时，不要先重装客户端，先检查 Key、路径、模型名和余额。

## OpenClaw 配置入口

OpenClaw 不只需要 `baseUrl` 和 `apiKey`，还要按官方结构配置 `models.providers`、`auth.profiles` 和 `agents.defaults`。

最小判断规则：

- 中转站支持 `/v1/responses`：使用 `api: "openai-responses"`。
- 只支持 `/v1/chat/completions`：使用 `api: "openai-completions"`。
- 要开启思考模式：模型条目加 `reasoning: true`，然后在 UI 或 `/reasoning` 打开当前会话 Think 档位。

完整示例见：[OpenClaw API 中转站配置指南](/openclaw-api-config)。

## 排错优先级

1. **401**：Key、Header、Bearer Token 或字段名。
2. **404 / model not found**：模型名、provider、路径或权限。
3. **429**：并发、余额、速率限制或上游限流。
4. **timeout**：本地网络、线路波动、代理冲突或客户端超时过短。
5. **配置无变化**：旧进程仍在，或客户端读取了另一个配置文件。

## 生产环境建议

- 不要让多个服务共用一个 Key。
- 为自动化任务设置预算和速率限制。
- 日志只记录必要的错误码、模型名、耗时和 token 用量，避免记录敏感 prompt。
- 对长任务设置超时和最大重试次数，避免故障时无限消耗余额。
