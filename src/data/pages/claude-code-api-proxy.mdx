# Claude Code 中转站手动配置教程

当自动安装或 ccswitch 导入不可用时，再使用手动配置。手动配置的核心是修改 Claude Code 的 `settings.json`，把 API Key 和 Base URL 写到 `env` 里。

> 先备份，再修改。文件里已有内容时，不要整文件清空，只把需要的字段合并进去。

## 适合谁看

- ccswitch 保存了，但 Claude Code 不生效。
- 想确认 Claude Code 到底读取了哪个 Key 和 Base URL。
- 需要手动调整上下文压缩阈值。

## 你需要准备

| 信息 | 从哪里拿 | 示例 |
|---|---|---|
| API Key | 账户页面 | `sk-xxxxxxxxxxxxxxxx` |
| Base URL | 本站统一地址 | `https://www.531288.xyz` |
| 压缩阈值 | 自行决定 | `78` 或 `80` |

本站统一使用 `https://www.531288.xyz`。如果服务地址后续调整，会在官网同步更新。

## 找到 settings.json

### Windows

```text
%USERPROFILE%\.claude\settings.json
```

可以按 `Win + R`，输入 `%USERPROFILE%\.claude` 后回车。

### macOS / Linux

```text
~/.claude/settings.json
```

Finder 可按 `Cmd + Shift + G`，输入 `~/.claude`。

如果没有 `.claude` 文件夹，可以先启动一次 Claude Code，或者手动创建文件夹和 `settings.json`。

## 写入配置

如果 `settings.json` 是空文件或不存在，可以写成：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "sk-xxxxxxxxxxxxxxxx",
    "ANTHROPIC_BASE_URL": "https://www.531288.xyz",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "78"
  }
}
```

如果文件里已有其他配置，只合并 `env` 字段：

```json
{
  "env": {
    "已有配置": "保留",
    "ANTHROPIC_API_KEY": "sk-xxxxxxxxxxxxxxxx",
    "ANTHROPIC_BASE_URL": "https://www.531288.xyz",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "78"
  }
}
```

## 保存并验证

1. 保存 `settings.json`。
2. 完全关闭 Claude Code 和当前终端。
3. 重新打开终端。
4. 执行 `claude`。
5. 发送测试消息：

```text
请回复“配置已生效”。
```

能收到回复就完成。如果提示认证失败，优先检查 Key；如果提示连接失败，优先检查 Base URL 和网络。

## 排错清单

| 现象 | 可能原因 | 处理方式 |
|---|---|---|
| Unauthorized / 认证失败 | Key 错了或多了空格 | 重新复制 API Key |
| Connection Error / 连接失败 | Base URL 错或网络不可用 | 确认地址为 `https://www.531288.xyz`，URL 末尾不要乱加 `/` |
| 保存后没变化 | 改错文件 | 确认路径是 `.claude/settings.json` |
| 设置不生效 | 旧进程还在 | 完全退出 Claude Code 和终端 |
| JSON 报错 | 少逗号或多逗号 | 用 JSON 校验工具检查格式 |

## 和上下文压缩的关系

`CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` 用来调整自动压缩触发阈值。推荐值 `78` 或 `80`，表示上下文占用到约 78%/80% 时提前压缩，避免撑到极限才处理。

压缩后可能出现：

- 早期细节变模糊。
- 缓存重新建立。
- 下一次请求费用变化。
- 需要重新补充关键背景。

长期项目规则建议写进 `CLAUDE.md`，不要只放在聊天里。
